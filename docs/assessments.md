# Assessments and Grading Rubric

## Overview
This page outlines the grading rubric for assignments and the capstone project, detailing how student performance will be evaluated throughout the course.

## Assignment Grading Rubric

### Weekly Assignments (70% of total grade)

#### Technical Implementation (40%)
- **Excellent (90-100%)**: Code is well-structured, efficient, and demonstrates deep understanding of concepts.
- **Good (80-89%)**: Code implements required functionality with minor issues.
- **Satisfactory (70-79%)**: Code implements basic functionality with some errors.
- **Needs Improvement (60-69%)**: Code has significant issues or incomplete implementation.
- **Unsatisfactory (&lt;60%)**: Code does not meet basic requirements or is incomplete.

#### Documentation and Report (25%)
- **Excellent (90-100%)**: Clear, thorough documentation with proper explanations and analysis.
- **Good (80-89%)**: Good documentation with minor omissions.
- **Satisfactory (70-79%)**: Basic documentation with adequate explanations.
- **Needs Improvement (60-69%)**: Limited documentation or unclear explanations.
- **Unsatisfactory (&lt;60%)**: Poor or missing documentation.

#### Understanding of Concepts (25%)
- **Excellent (90-100%)**: Demonstrates deep understanding of underlying concepts.
- **Good (80-89%)**: Shows good understanding of concepts with minor gaps.
- **Satisfactory (70-79%)**: Shows basic understanding of key concepts.
- **Needs Improvement (60-69%)**: Limited understanding of concepts.
- **Unsatisfactory (&lt;60%)**: Significant gaps in concept understanding.

#### Code Quality (10%)
- **Excellent (90-100%)**: Well-commented, efficient, and follows best practices.
- **Good (80-89%)**: Mostly well-written code with good practices.
- **Satisfactory (70-79%)**: Adequate code with some good practices.
- **Needs Improvement (60-69%)**: Code has significant quality issues.
- **Unsatisfactory (&lt;60%)**: Poor code quality with many issues.

### Quizzes (30% of total grade)

#### Conceptual Understanding (60%)
- **Excellent (90-100%)**: Demonstrates comprehensive understanding of concepts.
- **Good (80-89%)**: Shows good understanding with minor gaps.
- **Satisfactory (70-79%)**: Shows basic understanding of concepts.
- **Needs Improvement (60-69%)**: Limited understanding of concepts.
- **Unsatisfactory (&lt;60%)**: Significant gaps in understanding.

#### Problem Solving (40%)
- **Excellent (90-100%)**: Correctly solves complex problems with clear reasoning.
- **Good (80-89%)**: Solves problems correctly with minor issues.
- **Satisfactory (70-79%)**: Solves basic problems with some errors.
- **Needs Improvement (60-69%)**: Attempts to solve problems but with significant errors.
- **Unsatisfactory (&lt;60%)**: Unable to solve basic problems.

## Module-Specific Assessments

### [Module 1](modules/module-1.md#module-1-fundamentals-of-robotics)
- **Assignment Focus**: Basic robot programming and simulation.
- **Key Evaluation Points**: Code structure, basic functionality, understanding of robot components.
- **Weight**: 10% of total course grade.

### [Module 2](modules/module-2.md#module-2-robotic-sensing-and-perception)
- **Assignment Focus**: Sensor integration and data processing.
- **Key Evaluation Points**: Sensor fusion, perception algorithms, data analysis.
- **Weight**: 15% of total course grade.

### [Module 3](modules/module-3.md#module-3-robotic-motion-and-control)
- **Assignment Focus**: Kinematics, trajectory planning, and control.
- **Key Evaluation Points**: Motion planning algorithms, control system design.
- **Weight**: 15% of total course grade.

### [Module 4](modules/module-4.md#module-4-advanced-topics-and-applications)
- **Assignment Focus**: Advanced applications and AI integration.
- **Key Evaluation Points**: Implementation of AI/ML techniques, complex system integration.
- **Weight**: 15% of total course grade.

## Capstone Project Assessment

### Project Proposal (10% of total grade)
- **Problem Definition (30%)**: Clear problem statement and objectives.
- **Approach (40%)**: Feasible and well-researched technical approach.
- **Timeline and Resources (20%)**: Realistic timeline and resource planning.
- **Presentation (10%)**: Clear and professional presentation.

### Mid-term Checkpoint (5% of total grade)
- **Progress Assessment**: Implementation progress and milestone achievement.
- **Technical Challenges**: Identification and addressing of technical challenges.
- **Plan Adjustment**: Ability to adapt approach based on results.

### Final Capstone Project (25% of total grade)
- **Technical Implementation (35%)**: Quality and complexity of the implemented solution.
- **Functionality (25%)**: Degree to which the solution addresses the stated problem.
- **Documentation (20%)**: Code documentation, user manual, technical report.
- **Presentation (10%)**: Oral presentation and demonstration.
- **Team Collaboration (10%)**: For team projects, evaluation of collaboration.

### Capstone Rubric Details

#### Technical Implementation
- **Outstanding (90-100%)**: Innovative approach, handles edge cases, exceeds requirements.
- **Excellent (80-89%)**: Solid implementation that meets all requirements.
- **Good (70-79%)**: Implementation with minor issues or limitations.
- **Satisfactory (60-69%)**: Basic implementation with significant limitations.
- **Unsatisfactory (&lt;60%)**: Incomplete or non-functional implementation.

#### Functionality
- **Outstanding (90-100%)**: Fully functional, exceeds expectations, robust.
- **Excellent (80-89%)**: Fully functional, meets all requirements.
- **Good (70-79%)**: Mostly functional with minor issues.
- **Satisfactory (60-69%)**: Partially functional, meets basic requirements.
- **Unsatisfactory (&lt;60%)**: Not functional or significantly incomplete.

## Late Submission Policy
- Up to 24 hours late: 10% penalty.
- 24–48 hours late: 25% penalty.
- More than 48 hours late: Not accepted without prior approval.

## Academic Integrity
All work must be original. Collaboration is encouraged for learning concepts, but individual assignments must be completed independently. Code must be properly cited when using external sources or libraries.

## Grade Scale
- A: 90–100%
- B: 80–89%
- C: 70–79%
- D: 60–69%
- F: Below &lt;60%

## Extra Credit Opportunities
Extra credit may be available for:
- Advanced implementation beyond requirements.
- Creative solutions to problems.
- Contributing to open-source robotics projects.
- Presenting research in robotics.
